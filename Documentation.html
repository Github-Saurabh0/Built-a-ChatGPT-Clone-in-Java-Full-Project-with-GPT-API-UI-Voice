1. Introduction

This documentation provides detailed instructions on how to set up, configure, and test the ChatGPT Clone in Java project. The project demonstrates how to build a ChatGPT-like AI assistant using Java 17, OpenAI API, and a clean, modular approach. It provides a simple way to interact with OpenAI's GPT models through a Java application with both a console interface and a graphical user interface (GUI) with text-to-speech capabilities.

2. Project Overview

The ChatGPT Clone in Java project includes the following key components:

  • OpenAI API Integration: Connect to OpenAI's GPT models (GPT-3.5-turbo, GPT-4)
  • Graphical User Interface: A user-friendly interface built with Java Swing
  • Text-to-Speech: Voice output using FreeTTS
  • Conversation Management: Maintain conversation context for more natural interactions
  • Mock Response System: Test the application without an actual OpenAI API key

3. Setup and Installation

Prerequisites

  • Java 17 or higher
  • Maven
  • OpenAI API key (optional for testing)

Installation Steps

  1. Clone or download the repository to your local machine
  2. Navigate to the project directory
  3. Build the project using Maven: 
    mvn clean install

4. Configuring the OpenAI API Key

The application requires an OpenAI API key to connect to the GPT models. You have two options for configuring the API key:

Option 1: Using the config.properties File

  1. Open the config.properties file located in the root directory of the project
  2. Replace the placeholder value with your actual OpenAI API key: 
    # OpenAI API Configuration
# Replace with your actual API key
openai.api.key=your-actual-api-key-here
  3. Save the file

Option 2: Using Environment Variables

Alternatively, you can set the API key as an environment variable:

  • Windows: 
    set OPENAI_API_KEY=your-actual-api-key-here
  • Linux/macOS: 
    export OPENAI_API_KEY=your-actual-api-key-here

Testing Without an API Key

The application includes a mock response system that allows you to test the functionality without an actual OpenAI API key. If you leave the placeholder value (sk-your-api-key-here) in the config.properties file, the application will use simulated responses for testing purposes.

5. Running the Application

Using the Batch File (Windows)

The easiest way to run the application is to use the provided batch file:

  1. Double-click on run-ui.bat in the project directory
  2. The application will compile and launch automatically

Using Maven Commands

Alternatively, you can run the application using Maven commands:

mvn exec:java -Dexec.mainClass="com.chatgpt.clone.ui.ChatGPTApp"

6. Testing the Application

Testing with the Placeholder API Key

If you're using the placeholder API key (sk-your-api-key-here), the application will display a warning message and use simulated responses. This is useful for testing the UI and basic functionality without consuming your OpenAI API quota.

The mock response system provides different responses based on the content of your messages:

  • Messages containing "hello" or "hi" will receive a greeting
  • Messages containing "how are you" will receive a status response
  • Messages containing "weather" will receive a weather-related response
  • Messages containing "name" will receive an identity response
  • Messages with multiple question marks will receive a special response
  • Messages ending with a question mark will receive a generic question response
  • All other messages will receive a default response

Testing with a Valid API Key

To test the application with actual OpenAI responses:

  1. Configure your OpenAI API key as described in the Configuring the OpenAI API Key section
  2. Run the application
  3. The application will connect to the OpenAI API and provide real AI-generated responses

Testing the Voice Feature

The application includes text-to-speech functionality using FreeTTS:

  1. Launch the application
  2. Ensure the "Enable Voice" checkbox is selected
  3. Send a message and the response will be spoken aloud

Note: If the voice feature fails to initialize (e.g., due to missing dependencies or system compatibility issues), it will be automatically disabled, but the application will continue to function without voice output.

7. Features and Usage

User Interface

The graphical user interface includes the following components:

  • Chat Area: Displays the conversation history
  • Input Field: Enter your messages here
  • Send Button: Send your message to the AI
  • Clear Chat Button: Clear the conversation history
  • Model Selector: Choose between GPT-3.5-turbo and GPT-4 models
  • System Message: Customize the AI's behavior with a system message
  • Enable Voice: Toggle text-to-speech functionality

Conversation Management

The application maintains the conversation context, allowing for more natural and contextual interactions. For example, you can ask follow-up questions without repeating the entire context.

Model Selection

You can choose between different GPT models:

  • GPT-3.5-turbo: Faster and more cost-effective
  • GPT-4: More capable but potentially slower and more expensive

8. Troubleshooting

API Key Issues

  • Error Message: "API key not found"
    • Solution: Ensure your API key is correctly set in either the config.properties file or as an environment variable
  • Error Message: "Invalid API key"
    • Solution: Verify that your API key is correct and has not expired

Voice Feature Issues

  • Issue: Voice output not working
    • Solution: Check that the "Enable Voice" checkbox is selected. If the issue persists, the voice feature may not be compatible with your system. The application will continue to function without voice output.

Connection Issues

  • Error Message: "Error communicating with OpenAI API"
    • Solution: Check your internet connection and verify that the OpenAI API is accessible from your network

Other Issues

If you encounter any other issues, check the console output for error messages that may provide more information about the problem.
